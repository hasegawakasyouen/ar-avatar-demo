# ririka_kaihen.vrm 衣装混在・表情破綻の修正 設計書

## 背景

`ririka_kaihen.vrm`（[2026-07-12のVRM化](2026-07-12-ririka-kaihen-vrm-export-design.md)の成果物）をdesktop-vrm-mascotで実際に使用したところ、ユーザーの実見確認で以下の問題が判明した。

1. **衣装の混在**: 意図した黒猫悪夢衣装に、別の衣装（デフォルト衣装系）のメッシュが混ざって同時表示されている。腰・背中には本来存在しないはずの羽状パーツも表示されている
2. **表情の破綻**: 口が開きっぱなしになり、目にも意図しない模様が乗っている

前回のVRM化では小道具46個の非表示化・テクスチャ再リンク・幽霊発光修正まで行ったが、衣装メッシュの取捨選択とシェイプキーの静的な焼き込み値については未検査だった。

## ゴール

ユーザー提供のリファレンス画像（VRChat内スクリーンショット、黒猫悪夢衣装の正しい状態）を再現する:

- **衣装**: 黒猫悪夢衣装のみ（猫耳＋肉球グローブ＋黒レースドレス＋黒レースソックス）。混入している別衣装メッシュと腰の羽は除去（ユーザー確認済み: 羽は本来存在しないパーツ）
- **表情**: ニュートラルで口が閉じた落ち着いた状態。加えてdesktop-vrm-mascotの環境反応が使う5表情（happy / relaxed / surprised / sad / crying）が正常に動作すること

**完成判定はユーザーの目視承認とする**（前回の教訓: エージェントの「低優先度」判定だけでマージ可と結論づけず、ユーザー実見を挟む）。

## 方針（承認済み: 方針A）

既存の`source/ririka_kaihen.blend`生成パイプライン（`prepare_ririka_kaihen_blend.py`）を起点に修正する。Unity側の再ベイクは行わない。ただし下記エスカレーション条件に該当した場合のみUnity再ベイク（方針B）へ切り替える。

## 調査ステップ（実装の最初のタスク）

`source/ririka_kaihen.blend`に対し調査スクリプトを実行し、以下をダンプする:

1. **全メッシュオブジェクトの一覧** — 名前・`hide_viewport`状態・ポリゴン数・バウンディングボックスのおおよその位置。リファレンス画像と突き合わせ、「隠すべき混入メッシュ」（別衣装分・羽）を特定する
2. **シェイプキーのデフォルト値が0でないものの一覧** — 全メッシュを対象。口が開きっぱなし・目の模様の原因特定（`vrc.v_aa`等のビゼム系が1.0で残っている可能性が最有力）

## 修正設計

- `prepare_ririka_kaihen_blend.py`の既存`PROP_MESH_NAMES_TO_HIDE`リストに、調査で特定した混入メッシュ名を追加する（既存パターンの延長。新しい仕組みは作らない）
- 同スクリプトに「**Basis以外の全シェイプキーのデフォルト値（`value`）を0にリセットする**」ステップを新規追加する。表情はVRMのexpressionバインドが動的に制御するため、静的な焼き込み値は全て0が正しい
- FBX→blend→VRMのパイプラインを最初から再実行する（blendファイルを手動編集せず、スクリプトのみで再現可能な状態を保つ）
- `export_vrm_ririka_kaihen.py`（表情マッピング・ボーンマッピング）は、調査結果が変更の必要を示さない限り変更しない

## 検証

- `vrm-preview.html`でのブラウザ実機確認（**キャッシュバスティング`?t=Date.now()`必須** — 前回、古いキャッシュを見て誤判定した実績あり）:
  - ニュートラルで口が閉じている・目に余計な模様がない
  - 衣装がリファレンス画像と一致し、羽が消えている
  - 5表情（happy / relaxed / surprised / sad / crying）を実際に発火させて目視確認
- 最後にdesktop-vrm-mascotで読み込み、`window.__debugFlashExpression`で実環境確認
- 各段階のスクリーンショットをユーザーに提示し、最終的な完成判定はユーザーの目視承認で行う

## エスカレーション条件（→方針B: Unity再ベイク）

調査で以下のいずれかが判明した場合のみ、Unity側でlilycalinventoryの衣装トグルを正しい状態にしてNDMF Manual Bake→Binary FBX再書き出しからやり直す方針へ切り替える:

- (a) あるべき黒猫悪夢衣装のメッシュがFBXにそもそも存在しない
- (b) 複数衣装が1つのメッシュオブジェクトに結合されており、オブジェクト単位の表示切替では分離できない

（現VRMに黒レースドレスは表示されているため、(a)の可能性は低いと見込む）

## スコープ外

- desktop-vrm-mascot側のコード変更（一切なし）
- 太もものテクスチャ問題（前回「未解決」として記録）: 新しいリファレンス画像では太ももは素肌＋膝下に黒レースソックスに見えるため、「実は正しかった」可能性が高い。調査時に画像と照合し、一致していればクローズとして記録する
- 元のリリカ本体パイプライン（`export_vrm.py`/`export_vrma.py`）への変更
