## 背景・目的

現状、`README.md`のStep 2（骨格ストリップ）とStep 3内の高ポリゴン対応（デシメート）はすべてBlender GUIでの手作業として文書化されている。これを1本のスクリプトで自動化し、将来アバターを差し替える際の手間を減らす。

なお、テクスチャ再リンク（旧Step 4の課題）は別の並行作業（コミット `8dbe2f6`）で既に `scripts/convert_to_web.py` 側に自動化済みであり、今回の対象外。

## 前提・制約

- 既存の `scripts/convert_to_web.py`（Mixamoダウンロード後のGLB/USDZ変換・テクスチャ再リンク・アニメーション合成）には一切変更を加えない
- Mixamoへのアップロード・ダウンロード自体はWebサービスのため自動化しない（引き続き手動）
- テクスチャの自動リサイズ・GUIツール化は今回のスコープ外（YAGNI）

## 新規スクリプト: `scripts/prepare_for_mixamo.py`

**CLI:** `blender --background --python scripts/prepare_for_mixamo.py -- <入力FBX> <出力FBX>`

**定数（スクリプト冒頭）:**
```python
DECIMATE_THRESHOLD = 30000  # このトライアングル数を超えたらデシメート
DECIMATE_TARGET = 25000     # デシメート後の目標トライアングル数
```

**処理手順:**
1. 入力FBXをインポート
2. Shape Keys（ブレンドシェイプ）があれば全メッシュから削除する（表情アニメは本用途で不要なため）
3. 全メッシュオブジェクトを1つにJoin
4. 統合後のトライアングル数が `DECIMATE_THRESHOLD` を超えていれば、`DECIMATE_TARGET` になるようRatioを計算してDecimate Modifierを追加・適用する（閾値以下ならこのステップはスキップし、ログにその旨を出力する）
5. Armature ModifierとArmatureオブジェクトを削除する（メッシュのみの状態にする）
6. メッシュのみのFBXとして出力する

`--target-tris`等のCLIオプションは設けない。閾値・ターゲット値はスクリプト冒頭の定数で十分と判断する（YAGNI）。

## 動作確認方法

1. 過去に使用した高ポリゴンテストデータ（166,702トライアングル・骨格つき）で実際に実行し、デシメート・骨格ストリップが動作することを確認する
2. 出力FBXがBlenderで開けること、骨格が無くメッシュのみであること、トライアングル数が約25,000であることを確認する
3. 低ポリゴン（閾値以下）のダミーメッシュでも試し、デシメートがスキップされ骨格ストリップだけ行われることを確認する
4. 既存の `scripts/convert_to_web.py` には変更を加えないため、そちらのスモークテストの再実行は不要（影響範囲外）
